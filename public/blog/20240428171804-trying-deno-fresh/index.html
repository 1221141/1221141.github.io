trying deno fresh

/deno/ /fresh/ /svelte/ /web/

2024-04-28T17:18:04

I saw a video about https://deno.fresh.dev and thought it was interesting. Decided to write a bookmark manager for it.
I started yesterday and was able to write it in a simple way. The main benefit I see compared with sveltekit is that i don't need so many files in order to write the endpoint.
In sveltekit file routing one needs to have different files for different purposes (.svelte, .server.ts and .page.server.ts, page.ts)
In fresh you write only tsx files. The controller logic is in a handler function and you also return your javascript from it. using jsx.
The main selling point of the .svelte file is that you write html and css separately, but in reality nowadays people use inline style with tailwindcss. and the jsx is ok since it is server sided rendered.
It was a nice experience, but i'm not sure if well suited for large applications.
I think i will continue writing this bookmark app in fresh and see how it goes. All files are saved in markdown so it will be easy to migrate to another stack if needed in the future.
